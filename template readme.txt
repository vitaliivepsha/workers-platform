Сборка работает на webpack, стили пишутся на scss(обратно совместим с css, так что проблем возникнуть не должно).

При первой установке:
1) https://nodejs.org/uk/ устанавливаем node.js 8.10.0
2) https://www.python.org/downloads/ устанавливаем python 2.7.14
3) открыть консоль с правами администратора и запустить: npm install --global --production windows-build-tools
4) npm i && npm run dev

Перед первым запуском как обычно выполняем npm install в директории проекта.
Команды по запуску сборщика есть в файле README.md, тебе будет достаточно npm i && npm run dev
Настройки сборщика находятся в /config/webpack/global.js, менять там обычно ничего не надо, разве что в случае добавления php логики.
Исходники находятся в /app/assets
В /app/modules находятся наши модули, которые кочуют из проекта в проект, они реализуют функционал написанный ранее нами и простое подключение и инициализацию сторонних библиотек
при помощи добавления класов и дата-атрибутов в html. Периодически добавляю новые и дорабатываю существующие модули, но основной набор для ленда уже сформировался 
(jquery, slick, lightgallery, magnific-popup, fancybox, fancyselect, jscrollpane, jslider, chosen, наш скрипт для отправки и валидации форм, заготовки css классов)
В /app/app.js подключаются все модули, при необходимости пишется своя js логика (обычно хватает 5-10 строк на ленд, для решения специфических задач)
Основной шаблон html-структуры /app/assets/templates/layouts/index.html
В /app/assets/stylesheets находятся файлы стилей, app.css содержит базовые стили для всего проекта и подключение остальных scss файлов, _variables.scss - переменные, _fonts.scss - шрифты,
остальные файлы содержат стили для каждой секции страницы соответственно, при добавлении новых секций нужно создавать файл с её названием и подключать его в app.scss
_mixin.scss содержит scss логику, примеры:
@include bp(lg) {} заменяет @media only screen and (max-width: 1199px) {}
@include centering-y; - центрирует объект по вертикали

Информация по инициализации форм находится в /app/modules/forms/README.md, примеры в /app/modules/forms/index.html
При запуске сборщика страница доступна по адресу http://localhost:88 Необходимо проследить чтобы этот порт был свободен или изменить его в /config/webpack/global.js и package.json
